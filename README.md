## 方法论框架一览

```
核心总纲：实事求是  —— 一切从实际出发，不假设、不编造
        │
        ├─ 认知层：融会贯通 · 见微知著 · 提纲挈领 · 逆向思维
        ├─ 行动层：灵活运用 · 知行合一 · 举一反三 · 有的放矢
        ├─ 处世层：因地制宜 · 与时俱进 · 循序渐进
        └─ 全局层：统筹兼顾 · 张弛有度 · 大处着眼 · 大道至简
        │
        └─ 贯穿始终：互相印证  —— 任何判断必须多源交叉验证，不依赖单一来源
```

---

## 核心命题

> **培养多个 AI，是为了多一双看得见你盲区的眼睛。**

- 使用 AI = 给指令（工具思维）；培养 AI = 给判断力（伙伴思维）。
- 单一 AI 只会附和你；多个有思维的 AI 互相碰撞，才能突破一个人的判断力天花板。
- 方法论注入必须**到达行为层**（变成硬约束），不能停在知识层（只是"知道"）。

---

## 适用场景

- 做重大决策前，用"四性"（全面 / 正确 / 准确 / 完整）自检。
- 让 AI 产出结论时，强制"互相印证"——至少两个独立来源交叉验证。
- 设计 AI 协作系统时，把约束写进系统文件（如 `SOUL.md`），而非只留在提示词里。
